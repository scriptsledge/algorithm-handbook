## üìù Problem Description

!!! info "Problem Statement"
    Given an array `nums` of integers, return how many of them contain an **even number of digits**.

### Examples

**Example 1:**
```text
Input: nums = [12,345,2,6,7896]
Output: 2
Explanation: 
12 contains 2 digits (even number of digits). 
345 contains 3 digits (odd number of digits). 
2 contains 1 digit (odd number of digits). 
6 contains 1 digit (odd number of digits). 
7896 contains 4 digits (even number of digits). 
Therefore only 12 and 7896 contain an even number of digits.
```

**Example 2:**
```text
Input: nums = [555,901,482,1771]
Output: 1 
Explanation: 
Only 1771 contains an even number of digits.
```

### Constraints
*   `1 <= nums.length <= 500`
*   `1 <= nums[i] <= 10^5`

## üß† Intuition

The goal is to count how many numbers in the given array have an even number of digits. Since the maximum value of any number in `nums` is $10^5$, we can easily determine the number of digits by checking the range the number falls into.

## üí° Approach

1.  Initialize a `count` variable to 0.
2.  Iterate through each number `num` in the `nums` array.
3.  Check if `num` has an even number of digits:
    *   2 digits: $10 \le \text{num} \le 99$
    *   4 digits: $1000 \le \text{num} \le 9999$
    *   6 digits: $100000$ (since $10^5$ is the maximum value)
4.  If any of these conditions are met, increment `count`.
5.  Return the final `count`.

## üìä Complexity Analysis

*   **Time Complexity:** $O(N)$
    *   We traverse the array `nums` exactly once, where $N$ is the number of elements in the array.

*   **Space Complexity:** $O(1)$
    *   We use only a few extra variables (`count` and `num`), regardless of the input size.
